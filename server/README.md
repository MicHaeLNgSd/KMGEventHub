# KMG Event Manager - Server

Backend для системи управління офлайн-заходами (настільні ігри, конференції, зустрічі).

## Структура бази даних

### Таблиця `users`
Інформація про користувачів системи:
- **id** - унікальний ідентифікатор
- **phone_number** - номер телефону (обов'язковий, унікальний)
- **full_name** - повне ім'я
- **nickname** - прізвисько (унікальне)
- **age** - вік
- **photo_url** - URL фото профілю
- **bio** - біографія/опис
- **email** - електронна пошта (унікальна)
- **password_hash** - хеш пароля
- **is_active** - активний акаунт
- **created_at** - дата створення
- **updated_at** - дата оновлення

### Таблиця `events`
Інформація про заходи:
- **id** - унікальний ідентифікатор
- **creator_id** - ID організатора (посилання на users)
- **title** - назва заходу
- **description** - опис заходу
- **location** - місце проведення
- **latitude** - географічна широта
- **longitude** - географічна довгота
- **photo_url** - URL фото заходу
- **event_date** - дата та час проведення
- **max_participants** - максимальна кількість учасників
- **status** - статус (upcoming, ongoing, completed, cancelled)
- **created_at** - дата створення
- **updated_at** - дата оновлення

### Таблиця `event_participants`
Реєстрація учасників на заходи:
- **id** - унікальний ідентифікатор
- **event_id** - ID заходу
- **user_id** - ID користувача
- **status** - статус (registered, confirmed, cancelled, attended)
- **registered_at** - дата реєстрації

## Налаштування

### 1. Встановлення залежностей

```bash
cd server
npm install
```

### 2. Налаштування .env файлу

Скопіюйте `.env.example` в `.env` та настройте за потребою:

```bash
cp .env.example .env
```

Убедитесь, що PostgreSQL запущен на вказаних в `.env` параметрах:
- POSTGRES_HOST=localhost
- POSTGRES_PORT=5432
- POSTGRES_USER=postgres
- POSTGRES_PASSWORD=postgres

### 3. Створення бази даних та таблиць

Запустіть міграцію для створення бази даних та таблиць:

```bash
npm run db:migrate
```

### 4. Заповнення вибірковими даними

Запустіть seed для заповнення бази даних прикладами:

```bash
npm run db:seed
```

### 5. Запуск сервера

Розробка (з автоперезавантаженням):
```bash
npm run dev
```

Продакшен:
```bash
npm start
```

Сервер буде запущений на `http://localhost:3000`

## API Endpoints

### Користувачі
- `GET /api/users` - отримати всіх користувачів
- `GET /api/users/:id` - отримати користувача за ID
- `GET /api/users/:userId/events` - отримати заходи, створені користувачем
- `GET /api/users/:userId/registrations` - отримати реєстрації користувача

### Заходи
- `GET /api/events` - отримати всі заходи
- `GET /api/events/:id` - отримати детальну інформацію про захід

### Здоров'я
- `GET /api/health` - перевірити з'єднання з БД

## Команди управління БД

```bash
# Міграція (створення БД та таблиць)
npm run db:migrate

# Seed (заповнення вибірковими даними)
npm run db:seed

# Reset (видалення БД та повторне створення з seed)
npm run db:reset
```

## Структура проекту

```
server/
├── db/
│   ├── config.js          # Конфіг з'єднання з БД
│   ├── schema.sql         # SQL схема таблиць
│   ├── migrate.js         # Скрипт міграції

├── package.json           # Залежності
├── .env                   # Конфіг (не комітити!)
├── .env.example           # Приклад .env

```
